# Тестирование исправлений pyStudyFlash

## Внесенные изменения

### 1. Исправлены проблемы с подключением MQTT
- Улучшена обработка ошибок подключения
- Добавлено автоматическое переподключение при потере соединения
- Исправлены параметры подключения (порт, таймаут)
- Добавлена валидация настроек перед подключением

### 2. Улучшена система управления ролями
- Автоматическое назначение первого клиента контроллером
- Правильная передача роли контроллера при отключении
- Логирование изменений ролей
- Синхронизация ролей между сервером и клиентами

### 3. Добавлены радиокнопки для выбора контроллера
- Визуальное отображение текущего контроллера
- Возможность переключения контроллера через UI
- Группировка радиокнопок для корректной работы

### 4. Улучшена обработка отключения клиентов
- Корректная очистка ресурсов при отключении
- Автоматическое удаление окон отключенных клиентов
- Передача роли контроллера при отключении активного контроллера

## Инструкции по тестированию

### Подготовка к тестированию

1. Убедитесь, что у вас установлены все зависимости:
   ```bash
   pip install -r requirements.txt
   ```

2. Настройте MQTT брокер (можно использовать публичный broker.hivemq.com)

3. Создайте настройки в файле `sets/settings.ini`:
   ```ini
   [General]
   server_address=test@example.com
   server_password=testpass
   mqtt_address=broker.hivemq.com
   mqtt_port=1883
   mqtt_timeout=60
   ```

### Тест 1: Подключение к серверу

1. Запустите сервер:
   ```bash
   python pystudyflash.py
   ```

2. Нажмите "Старт доступ" - должно появиться сообщение о успешном подключении

3. Запустите клиент:
   ```bash
   python pystudyflash.py
   ```

4. Введите адрес сервера и нажмите "Подключиться"
5. Проверьте, что клиент подключился и получил роль контроллера

### Тест 2: Многопользовательский режим

1. Запустите несколько клиентов (2-3 экземпляра)
2. Убедитесь, что только первый клиент получил роль контроллера
3. На сервере должны появиться окна для каждого клиента
4. Проверьте радиокнопки - активна должна быть только у контроллера

### Тест 3: Переключение контроллера

1. На сервере выберите другую радиокнопку в окне клиента
2. Убедитесь, что роль контроллера передалась выбранному клиенту
3. Проверьте, что у предыдущего контроллера роль изменилась на "Viewer"
4. Только новый контроллер должен иметь возможность управлять мышью/клавиатурой

### Тест 4: Отключение клиентов

1. Закройте один из клиентов (не контроллера)
2. Убедитесь, что его окно исчезло с сервера через несколько секунд
3. Закройте клиента-контроллера
4. Проверьте, что роль контроллера автоматически передалась другому клиенту

### Тест 5: Восстановление соединения

1. Временно отключите интернет на клиенте
2. Убедитесь, что клиент пытается переподключиться
3. Восстановите соединение
4. Проверьте, что клиент успешно переподключился

## Возможные проблемы и решения

### Проблема: Клиент не может подключиться
**Решение:** 
- Проверьте настройки MQTT брокера
- Убедитесь, что сервер запущен и подключен к MQTT
- Проверьте правильность адреса сервера

### Проблема: Не работает управление мышью/клавиатурой
**Решение:**
- Убедитесь, что клиент имеет роль контроллера
- Проверьте, что окно клиента активно
- На Windows может потребоваться запуск от администратора

### Проблема: Радиокнопки не переключают контроллера
**Решение:**
- Убедитесь, что клиенты подключены
- Проверьте консоль сервера на наличие ошибок
- Перезапустите сервер
